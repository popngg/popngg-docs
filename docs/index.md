# popn.gg 리팩토링 문서

이 문서는 popn.gg 서버 리팩토링을 위한 공용 컨텍스트입니다. 목표는 기존 프로젝트를 그대로 이어받아 기능 변경, 데이터 모델 재설계, API 재구현, 프론트 연동 방식을 정리하는 것입니다.

## 현재 저장소

- Java 17, Spring Boot 3.2.4, Gradle 멀티모듈 프로젝트입니다.
- 모듈은 `popngg-api`, `popngg-application`, `popngg-domain`, `popngg-infra`로 나뉩니다.
- 현재 주요 도메인은 유저, 차트, 플레이데이터, 히스토리입니다.
- 현재 API prefix는 일부 컨트롤러 기준 `/api/v2`입니다.

## 리팩토링 핵심 목표

- 게임 시스템 변경: 신규 랭크, 신규 메달, 갱신 코드, 자켓/장르 표시 정책 반영
- 데이터 변경: 비밀번호 salt 기반 해싱, FK 제거, `chart`와 `song` 메타데이터 분리, songhash 재설계
- 서버 재구현: 프론트에서 하던 데이터 가공 로직을 백엔드로 이동
- 문서화: LLM이 읽을 컨텍스트와 개발자가 볼 MkDocs 문서 유지

## 문서 운영 원칙

- 확정된 내용과 검토 중인 내용을 분리합니다.
- DB 변경은 마이그레이션 경로까지 같이 기록합니다.
- API는 프론트가 그대로 표시할 수 있는 응답 형태를 우선합니다.
- 애매한 정책은 `decisions.md`에 ADR 형태로 남깁니다.

## 로컬 문서 실행

```bash
pip install mkdocs-material
mkdocs serve
```

브라우저에서 `http://127.0.0.1:8000`을 열면 됩니다.
