# MVP 범위

## 포함

- 유저 등록/갱신/로그인
- 이메일 기반 비밀번호 복구
- 곡/채보 조회
- 플레이데이터 갱신
- 유저별 전체 플레이데이터 조회
- 유저 팝클 테이블
- 곡별 랭킹
- 레벨별 랭크/메달 집계
- 갱신 로그
- 플레이데이터 히스토리
- Jenkins + Docker 배포
- Flyway schema migration

## MVP 이후

- 검색 태그 기여
- 한국어 검색 태그 승인 플로우
- 이메일 인증 강화
- 상세 모니터링
- 관리자 데이터 검수 UI
- 게임 코드/표시 정책 DB 관리 UI
- strict judgement/gauge 데이터 확장

여기서 게임 코드/표시 정책은 랭크, 메달, 난이도, 크레딧 종류처럼 코드와 화면 표시명이 정해져 있는 기준값을 말합니다. MVP에서는 코드와 API 응답으로 제공하고, 운영자가 화면에서 직접 수정하는 DB 관리 UI는 만들지 않습니다.

## 출시 전략

1. 신규 스키마와 운영 기준을 먼저 확정합니다.
2. 레거시 코드는 마이그레이션 입력으로만 참고합니다.
3. Flyway로 baseline을 만들고, 데이터 이전은 별도 job/script로 분리합니다.
4. API를 MVP 범위로 먼저 공개합니다.
5. 운영 중 필요한 기능을 새 기준에 맞춰 작은 단위로 확장합니다.
