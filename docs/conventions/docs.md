# 문서 컨벤션

## 구조

- 기획 문서는 배경, 목표, 범위를 설명합니다.
- 설계 문서는 선택지, 결정 이유, 영향 범위를 설명합니다.
- API 문서는 OpenAPI와 사람이 읽는 설명을 함께 유지합니다.
- 운영 문서는 실제 배포/복구 절차를 기준으로 작성합니다.

## 작성 규칙

- 공개 가능한 정보만 작성합니다.
- 확정되지 않은 내용은 `초안`, `Proposed`, `Open` 등으로 표시합니다.
- 로컬 경로, 개인 이름, 비밀값, 실제 운영 credential은 쓰지 않습니다.
- 코드나 SQL 예시는 복사해서 사용할 수 있게 완전한 형태로 작성합니다.

## 문서 변경 기준

다음 변경은 문서 업데이트가 필요합니다.

- DB schema 변경
- API request/response 변경
- 인증/권한 정책 변경
- 배포 방식 변경
- 운영 runbook 변경
- 게임 시스템 정책 변경
